# Agents

## Cursor Cloud specific instructions

This is a Next.js 14 calisthenics workout tracker app ("CaliTrack") with TypeScript and Tailwind CSS. Data is persisted in localStorage (no backend/database required).

### Key commands

| Action | Command |
|--------|---------|
| Dev server | `npm run dev` (port 3000) |
| Build | `npm run build` |
| Lint | `npm run lint` |
| Tests | `npm test` |

### Architecture

- **Next.js App Router** — pages under `src/app/`
- **State** — React context (`src/context/WorkoutContext.tsx`) + localStorage (`src/lib/storage.ts`)
- **Data** — Static exercise and workout data in `src/data/`
- **Components** — Reusable UI in `src/components/`
- **Types** — Shared TypeScript types in `src/lib/types.ts`

### Non-obvious notes

- The app is fully client-side; all pages use `"use client"`. No API routes or server-side data fetching.
- `next-env.d.ts` is auto-generated by Next.js — do not edit it manually (it's also in `.gitignore`).
- Jest requires `ts-node` to parse `jest.config.ts`; it's included in devDependencies.
- The Timer and RestTimer components use `setInterval` — they rely on browser timers and won't work in SSR.
